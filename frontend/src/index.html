SAME-HOST SSH OVER USB

Recommended workflow. Connect the Pwnagotchi to the desktop via USB, wait for the USB/RNDIS network interface to come up, then use the IP that the device exposes on that mapped network here in TARGET IP.

WHAT TO VERIFY

Make sure the desktop can reach the Pwnagotchi over the USB-mapped network before testing. TEST CONNECTION validates SSH access and the configured remote path only; it does not configure the network for you.

If you need a reference for the host/user/path setup, check the Pwnagotchi project configuration docs and use the values exposed by your USB networking setup.

M5EVIL ADMIN WEBUI
REACHABILITY MODES
SAME-NETWORK WEBUI

Recommended workflow. Connect the Cardputer to the same Wi-Fi/network as the desktop, use the IP shown by the Cardputer here in M5EVIL HOST, then run Start Captive Portal and sign in with the configured WebUI credentials.

REMOTE EXPOSURE

UPnP NAT exposes internal port 80 on another reachable port like 50080. Reverse TCP Tunnel requires tcp_host= and tcp_port= in the device config.txt. If the device shows Error check TCP host in config file, your tcp_host is missing.

CONFIG OWNERSHIP

Device-side config.txt: wifi_ssid, wifi_password, tcp_host, tcp_port, webpassword. KOVIL MAP-side fields: host, port, username, password. TEST CONNECTION validates the endpoint configured here only; it does not configure UPnP NAT or Reverse TCP Tunnel for you.

Use webpassword=... in the device config.txt as the source of truth for the WebUI password. The preset keeps test as your current configured default, but firmware builds may differ.
