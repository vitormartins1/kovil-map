# Demo Mode & Mock Data

KOVIL MAP now ships an opt-in public demo pack called `showcase-core-v1`.

It exists for two main cases:

- onboarding new users who do not have capture devices yet
- generating screenshots or walkthroughs without exposing sensitive field data

The v1 design is a **safe switch**, not a merge. When installed, the demo pack temporarily replaces the mutable runtime dataset used by the app. When removed, KOVIL MAP restores the previous runtime snapshot if one exists.

## What Is Included

The pack is synthetic and versioned under `backend/demo_data/showcase-core-v1/`.

It includes:

- synthetic Rio de Janeiro Wardrive sessions
- handshake captures across Pwnagotchi, Bruce and M5Evil paths
- capture-scoped artifacts and a combined candidate example
- demo RAW Sniffer inputs for Bruce RAW, M5 RawSniffer and M5 Master Sniffer
- a small demo wordlist bundle exposed only while demo mode is active
- UI seed data for `Targets`, `Favorites`, and a few helpful panel modes

The data is designed to exercise:

- Tactical Map
- Recon Center
- WarDrive Workspace
- Raw Sniffer
- Batch / No-GPS workflows
- cracking demos with varied target difficulty

## How To Use It

Use the built-in maintenance controls inside the app:

1. Open `System Settings`.
2. Go to `Maintenance`.
3. Click `INSTALL DEMO DATA`.

KOVIL MAP will:

- snapshot the current mutable runtime dataset
- replace the active runtime roots with the showcase pack
- reload caches and data
- attempt a best-effort Recon prewarm when `tshark` is available

To leave demo mode:

1. Open `System Settings`.
2. Go to `Maintenance`.
3. Click `REMOVE DEMO DATA`.

If a real runtime snapshot existed before install, it is restored automatically.

## Notes

- Demo mode is **not enabled by default** in v1.
- Real and demo datasets are intentionally not mixed together in the active view.
- Demo wordlists do not overwrite the user's configured `custom_wordlists_path`.
- The tracked demo source pack is deterministic and can be regenerated by maintainers with:

```bash
cd backend
.venv313/bin/python -m app.tools.build_demo_data --validate
```
