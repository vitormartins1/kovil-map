# Manual Import Layout

This page documents the current local folder layout for manual imports.

Use this structure for new imports. Older fallback layouts may still be readable in some flows, but they are no longer the recommended way to add data.

## Automatic vs Manual Ingest

KOVIL MAP currently supports two ingestion styles:

- **Remote sync:** used for `Pwnagotchi` over SSH/SFTP and `M5Evil Cardputer` over `Admin WebUI` through `POST /api/sync`
- **Manual local import:** copy files into the local `backend/data/` structure and then refresh the app

Brucegotchi and M5Evil handshake captures are supported by the local catalog. Brucegotchi is still a manual-import workflow; `M5Evil Cardputer` can now be imported either manually or through the automated sync profile.

## Current Folder Layout

```text
backend/data/
  handshakes/
  BrucePCAP/
    handshakes/
    rawsniffer/
    .metadata/
  m5evil/
    handshakes/
    rawsniffer/
    .metadata/
  wardrive/
    manifest.json
    session_tags.json
    merged/
```

## Handshake Imports

### `backend/data/handshakes/`

Use this folder for classic Pwnagotchi-style handshake files, GPS sidecars, compatibility artifacts, and shared legacy outputs.

Typical contents:

- `MyWiFi_AABBCCDDEEFF.pcap`
- `MyWiFi_AABBCCDDEEFF.gps.json`
- `MyWiFi_AABBCCDDEEFF.geo.json`
- `MyWiFi_AABBCCDDEEFF.paw-gps.json`
- `MyWiFi_AABBCCDDEEFF.details`
- `MyWiFi_AABBCCDDEEFF.22000`
- `MyWiFi_AABBCCDDEEFF.try`
- `MyWiFi_AABBCCDDEEFF.cracked`
- `MyWiFi_AABBCCDDEEFF__wdrs__.22000`

Notes:

- this is the canonical home for classic handshake imports
- it still stores shared legacy sidecars used by the handshake-set catalog as compatibility fallback
- GPS sidecars placed here can enrich networks with coordinates during loading

### `backend/data/handshakes/captures/<capture_id>/`

This is the preferred home for new derived artifacts tied to one specific capture inside a handshake set.

Typical contents:

- `capture.details`
- `capture.22000`
- `capture.try`
- `capture.cracked`
- `manifest.json`

Notes:

- the source `.pcap` remains in its original family folder
- capture-scoped writes are used when cracking, conversion, fingerprint extraction, or Aircrack target a specific `capture_id`
- older shared sidecars in `backend/data/handshakes/` remain readable but are no longer the preferred write target

### `backend/data/handshakes/combined/<mac_clean>/<build_id>/`

This is the home for manual combined candidates built from multiple captures of the same BSSID.

Typical contents:

- `combined.22000`
- `combined.try`
- `combined.cracked`
- `manifest.json`

Notes:

- combined candidates are explicit crack targets
- they do not replace the preferred capture automatically

### `backend/data/BrucePCAP/handshakes/`

Use this folder for Brucegotchi handshake captures in the `HS_<MAC>.pcap` style.

Typical contents:

- `HS_AABBCCDDEEFF.pcap`

Notes:

- Brucegotchi handshake captures are grouped into handshake sets by normalized BSSID/MAC
- derived artifacts still live in `backend/data/handshakes/` in the current v1 model

### `backend/data/BrucePCAP/rawsniffer/`

Use this folder for new RAW Sniffer captures generated by the Bruce firmware.

Typical contents:

- `raw_1.pcap`
- `raw_capture_20260404.pcap`

Generated sidecars:

- `.metadata/*.json` for RAW metadata cache

Notes:

- `raw_*.pcap` belongs in `rawsniffer/`, not in `handshakes/`
- the RAW Sniffer panel works from this folder
- deleting a RAW file from the app also removes its cached metadata and sibling generated `.22000` when present

### `backend/data/BrucePCAP/`

The Bruce root remains readable as a legacy RAW fallback location.

Notes:

- older Bruce RAW imports stored directly in the root are still discovered
- new RAW imports should prefer `backend/data/BrucePCAP/rawsniffer/`
- `HS_<MAC>.pcap` can still be discovered in fallback flows, but `handshakes/` is the preferred home for new Brucegotchi handshake files

### `backend/data/m5evil/handshakes/`

Use this folder for M5Evil handshake captures.

Typical contents:

- `HS_AABBCCDDEEFF.pcap`

Notes:

- M5Evil uses the same `HS_<MAC>.pcap` naming family as Bruce-style captures
- M5Evil captures remain distinct in the handshake catalog because capture identity is not based only on basename
- derived artifacts still remain centralized in `backend/data/handshakes/` in the current implementation

### `backend/data/m5evil/rawsniffer/`

Use this folder for new M5Evil RAW Sniffer captures.

Typical contents:

- `RawSniff_00.pcap`
- `RawSniff_01.pcap`

Notes:

- RAW metadata cache for M5Evil lives in `backend/data/m5evil/.metadata/`
- these files appear in the RAW Sniffer UI grouped under `M5Evil`

### `backend/data/m5evil/mastersniffer/`

Use this folder for M5Evil `masterSniffer_*.pcap` captures.

Typical contents:

- `masterSniffer_0.pcap`
- `masterSniffer_1.pcap`

Notes:

- these files are treated as RAW captures, not as normal `HS_*.pcap` handshake captures
- they appear in the RAW Sniffer UI under `M5Evil` with subtype `MASTER SNIFFER`

## WarDrive Imports

### `backend/data/wardrive/`

Drop WiGLE-compatible CSV files here.

Typical contents:

- `session-1.csv`
- `session-2.csv`
- `manifest.json`
- `session_tags.json`
- `merged/merged-YYYYMMDD-HHMMSS.csv`

Notes:

- new CSVs dropped in the root folder are discovered as original sessions
- `manifest.json` controls which sessions are active vs ignored
- `session_tags.json` stores vehicle / transport tagging
- merged sessions are written to `merged/`
- M5Evil Cardputer auto-sync also writes Wardrive CSVs here, prefixed as `m5evil__<remote_filename>`

## Manual Import Workflow

### Handshakes

1. Copy the capture into the correct folder for its source family.
2. If you have GPS sidecars for classic Pwnagotchi-style captures, copy them into `backend/data/handshakes/` alongside the `.pcap`.
3. Click **Sync** in the app, or restart the backend if you want a clean reload.

### RAW Sniffer

1. Copy `raw_*.pcap` into the source-specific RAW folder:
   - Bruce firmware: `backend/data/BrucePCAP/rawsniffer/`
   - M5Evil `RawSniff`: `backend/data/m5evil/rawsniffer/`
   - M5Evil `masterSniffer`: `backend/data/m5evil/mastersniffer/`
2. Open the `RAW SNIFFER` panel.
3. Run extraction for the selected file or for pending files.
4. Optionally build RAW Analysis or inspect the file later from Cracking Operations for networks linked to that RAW capture.

### WarDrive

1. Copy WiGLE-compatible CSV files into `backend/data/wardrive/`.
2. Open the WarDrive workspace and refresh it, or call `POST /api/wardrive/refresh`.
3. Review sessions, tags, and merge outputs from the workspace.

## Practical Rules

- Use `backend/data/handshakes/` for classic Pwnagotchi-style captures and shared sidecars.
- Use `backend/data/BrucePCAP/handshakes/` for Brucegotchi handshakes.
- Use `backend/data/BrucePCAP/rawsniffer/` for new Bruce RAW captures.
- Use `backend/data/m5evil/handshakes/` for M5Evil handshakes.
- Use `backend/data/m5evil/rawsniffer/` for M5Evil `RawSniff_*.pcap`.
- Use `backend/data/m5evil/mastersniffer/` for M5Evil `masterSniffer_*.pcap`.
- Use `backend/data/wardrive/` for WiGLE CSV sessions.
- Prefer the current layout above for new imports rather than older fallback layouts.
